How Steganography Works

1. Cover Image

Any PNG or JPEG image. Each pixel has three colour channels — Red, Green, Blue — each stored as a number 0–255.

2. LSB Encoding

The message is converted to binary. The Least Significant Bit (last bit) of each colour value is replaced with one bit of the message. Changing a pixel from 200 to 201 is invisible to the human eye.

R=200 11001000 → 11001001 R=201

3. AES-256-GCM Encryption

When a passphrase is set, the message is encrypted with AES-256 in GCM mode before embedding. PBKDF2-SHA1 (100k iterations) derives the key from your passphrase + random salt, so brute-forcing is impractical.

Passphrase + Salt → PBKDF2 → Key → AES-GCM

4. Detection Resistance

LSB changes are statistically tiny. Without knowing a message is hidden, visual or histogram analysis rarely reveals its presence. Encryption adds another layer — the payload looks like random noise.

CTF Use Case

In Capture The Flag competitions, steg challenges hide flags inside image files. This tool was built after encountering multiple LSB steg challenges where custom tooling was needed.

Capacity

A 1920×1080 image has 2,073,600 pixels × 3 channels = 6,220,800 bits = ~777 KB of payload capacity. Encrypted overhead is only ~44 bytes (salt + nonce + tag).
